1. Master Module: Setup & FIFO

The master_ahb.v file acts as the brain. It takes simple instructions from the user (like "Write these 4 words starting at Address X") and translates them into strict AHB protocol signals. (Note: Stitching the code blocks in sections 1, 2, and 3 together creates your complete master_ahb.v file).

`timescale 1ns / 1ps

module master_ahb(
    // AHB SIGNALS
    input CLK_MASTER,
    input RESET_MASTER,
    input HREADY, 
    input [31:0] HRDATA,
    
    // USER DEFINED SIGNALS
    input [31:0] data_top,
    input write_top,       // 1 = Write, 0 = Read
    input [3:0] beat_length, 
    input enb,             // enb=1 master will start operation
    input [31:0] addr_top, // Starting address from TB
    input wrap_enb,        // 1 = WRAP burst, 0 = INCR burst
    
    // AHB OUTPUT SIGNALS
    output [31:0] HADDR,
    output reg HWRITE,
    output reg [2:0] HSIZE,
    output reg [2:0] HBURST,
    output reg [1:0] HTRANS,
    output reg [31:0] HWDATA,
   
    // USER DEFINE SIGNAL FIFO
    output fifo_empty, 
    output fifo_full
);
    
    // State machine parameters
    parameter idle                = 3'b000;
    parameter write_state_address = 3'b001;
    parameter write_state_data    = 3'b010;
    parameter read_state_address  = 3'b011;
    parameter read_state_data     = 3'b100;
    
    reg [2:0] present_state, next_state;
    reg [31:0] addr_internal;
    reg [3:0] count;
    integer i;
    
    // Wrap functionality registers
    reg [31:0] wrap_base;
    reg [31:0] wrap_boundary;
    
    // FIFO signals for Write Operations
    reg [3:0] rd_ptr, wr_ptr;
    reg [31:0] mem [14:0];
    reg [31:0] read_buffer [15:0]; 
    
    assign fifo_empty = (wr_ptr == rd_ptr);
    assign fifo_full = ((wr_ptr == 14 ? 4'd0 : wr_ptr + 1'b1) == rd_ptr); 
    
    // FIFO WRITE LOGIC (User pushing data into Master)
    always @(posedge CLK_MASTER) begin
        if(RESET_MASTER) begin
            for(i = 0; i < 15; i = i+1) mem[i] <= 32'd0;
            wr_ptr <= 0;
        end
        else if(write_top && !fifo_full && enb == 0) begin 
            mem[wr_ptr] <= data_top;
            wr_ptr <= (wr_ptr == 14) ? 4'd0 : wr_ptr + 1'b1;
        end
    end
How it works: The User Interface & FIFO

Why do we need a FIFO? In a high-speed system, the user (or testbench) might want to provide data faster than the bus can send it, especially if the Slave asks the Master to wait (HREADY = 0). By pushing user data into the mem array (the FIFO), the Master can safely store the data and pop it out onto the HWDATA bus precisely when the protocol allows.

FIFO Flags: fifo_empty and fifo_full prevent the user from underflowing or overflowing the memory array.

2. Master: FSM & Address Calculations

This sequential logic block updates the state machine and calculates the addresses. It handles the complex math required for wrapping bursts.

    // SEQUENTIAL LOGIC (State, Address, and Data Updates)
    always @(posedge CLK_MASTER) begin
        if(RESET_MASTER) begin
            present_state <= idle;
            count <= 0; rd_ptr <= 0;
            addr_internal <= 32'h0000_0000;
            wrap_base <= 32'd0; wrap_boundary <= 32'd0;
            for(i = 0; i < 16; i = i+1) read_buffer[i] <= 32'd0;
        end
        else begin
            present_state <= next_state; // Advance state machine
            
            // --- START OF TRANSACTION ---
            if (present_state == idle && enb) begin
                addr_internal <= addr_top;
                count <= 0;
                
                // Wrap Calculations
                if (wrap_enb) begin
                    wrap_boundary <= 32'd16; // 4 Beats x 4 Bytes = 16 Bytes
                    // Base Address = Starting Address & ~(15)
                    wrap_base <= addr_top & ~(32'd15); 
                end
            end 
            
            // --- WRITE DATA PHASE ---
            else if (present_state == write_state_data && HREADY) begin
                count <= count + 1'b1;
                rd_ptr <= (rd_ptr == 14) ? 4'd0 : rd_ptr + 1'b1;
                
                // Address incrementing logic
                if (count < beat_length - 1) begin
                    if (wrap_enb) begin
                        if ((addr_internal + 4) == (wrap_base + wrap_boundary))
                            addr_internal <= wrap_base;
                        else
                            addr_internal <= addr_internal + 4;
                    end else begin
                        addr_internal <= addr_internal + 4; // Normal INCR burst
                    end
                end
            end
            
            // --- READ DATA PHASE ---
            else if (present_state == read_state_data && HREADY) begin
                count <= count + 1'b1;
                read_buffer[count] <= HRDATA; // Store slave data
                
                // Address incrementing logic (Identical to Write Phase)
                if (count < beat_length - 1) begin
                    if (wrap_enb) begin
                        if ((addr_internal + 4) == (wrap_base + wrap_boundary))
                            addr_internal <= wrap_base;
                        else
                            addr_internal <= addr_internal + 4;
                    end else begin
                        addr_internal <= addr_internal + 4;
                    end
                end
            end
        end
    end
How it works: Calculating Wraps & Pipeline Steps

The FSM: The master transitions from idle -> address_phase -> data_phase. Notice that the state and address counters only update if (HREADY). If the slave pulls HREADY low, this entire block freezes, correctly executing a wait state!

Wrap Base Math: Look at wrap_base <= addr_top & ~(32'd15);. This is bitwise magic. 15 in binary is 01111. The NOT of that is 10000. By doing a bitwise AND with the starting address, we force the last 4 bits to 0000. This perfectly calculates the absolute base boundary of any 16-byte block.

The Loop: As the address increments by 4 (addr_internal + 4), it checks if it hits the ceiling (wrap_base + 16). If it does, instead of continuing to go up, it snaps back down to the wrap_base.

3. Master: Output Logic (Combinational)

This block constantly evaluates the current state and drives the AHB output signals. This is where HTRANS changes from NONSEQ to SEQ.

    // COMBINATIONAL LOGIC (Next State & Outputs)
    always @(*) begin
        // Default assignments to prevent latches
        next_state = present_state;
        HSIZE  = 3'b010; // 32-bit Data Width
        HBURST = 3'b000; // Default SINGLE
        HTRANS = 2'b00;  // Default IDLE
        HWRITE = 1'b0;
        HWDATA = 32'h0000_0000;
        
        // Logic to determine Burst Type
        if (beat_length == 4) begin
            if (wrap_enb) HBURST = 3'b010;      // WRAP4
            else          HBURST = 3'b011;      // INCR4
        end else begin
            HBURST = 3'b000;                    // SINGLE
        end

        case(present_state)
            idle: begin
                HTRANS = 2'b00; 
                if (enb) begin
                    if (write_top && !fifo_empty) 
                        next_state = write_state_address; 
                    else if (!write_top) 
                        next_state = read_state_address;  
                end
            end
                  
            write_state_address: begin
                HWRITE = 1'b1;
                HTRANS = 2'b10; // NONSEQ for the first beat
                if (HREADY) next_state = write_state_data;
            end
                
            write_state_data: begin
                HWRITE = 1'b1;
                HWDATA = mem[rd_ptr]; // Drive data bus from FIFO
                
                if (count < beat_length - 1) begin
                    HTRANS = 2'b11; // SEQ for subsequent beats
                    if (HREADY) next_state = write_state_data;
                end else begin
                    HTRANS = 2'b00; // IDLE after burst finishes
                    if (HREADY) next_state = idle;
                end
            end
            
            read_state_address: begin
                HWRITE = 1'b0; 
                HTRANS = 2'b10; // NONSEQ for the first beat
                if (HREADY) next_state = read_state_data;
            end
            
            read_state_data: begin
                HWRITE = 1'b0; 
                if (count < beat_length - 1) begin
                    HTRANS = 2'b11; // SEQ
                    if (HREADY) next_state = read_state_data;
                end else begin
                    HTRANS = 2'b00; // IDLE
                    if (HREADY) next_state = idle;
                end
            end
             
            default: next_state = idle;
        endcase
    end
    
    assign HADDR = addr_internal;
    
endmodule
How it works: Driving the Protocol

Default Values: We set defaults at the top of the block (like HSIZE = 3'b010) so that Vivado/Quartus doesn't create unwanted latches in our synthesized hardware.

The HTRANS Sequence: Look at write_state_address. HTRANS is explicitly set to 2'b10 (NONSEQ) because this is the first beat of a new burst. In the very next state (write_state_data), as long as we haven't reached the end of the burst (count < beat_length - 1), HTRANS changes to 2'b11 (SEQ).

HWDATA Mapping: In the data state, HWDATA is fed directly from mem[rd_ptr]. Because rd_ptr was advanced in the sequential block on the rising edge, fresh data naturally appears here every cycle.

4. AHB Slave Module (`slave_ahb.v`)

The slave is the component receiving the commands. To comply with the AHB pipeline rules, the slave must capture the control signals during the Address Phase, and act on them one clock cycle later in the Data Phase.

`timescale 1ns / 1ps

module slave_ahb(
    input HCLK,
    input HRESET,
    input [31:0] HADDR,
    input HWRITE,
    input [2:0] HSIZE,
    input [2:0] HBURST,
    input [1:0] HTRANS,
    input [31:0] HWDATA,
    output reg HREADY,
    output reg HRESP,
    output reg [31:0] HRDATA
);
    
    // Internal registers to capture the Address Phase
    reg [1:0] htrans_reg;
    reg hwrite_reg;
    reg [31:0] haddr_reg;
    
    // =========================================================
    // ADDRESS PHASE (Synchronous)
    // =========================================================
    always @(posedge HCLK) begin
        if (HRESET) begin
            htrans_reg <= 2'b00;
            hwrite_reg <= 1'b0;
            haddr_reg <= 32'd0;
        end else if (HREADY) begin
            // Sample control signals to use them in the next clock cycle
            htrans_reg <= HTRANS;
            hwrite_reg <= HWRITE;
            haddr_reg <= HADDR;
        end
    end

    // =========================================================
    // DATA PHASE (Combinational)
    // =========================================================
    always @(*) begin
        // Default assignments (Zero-wait state slave)
        HREADY = 1'b1;  // Always ready to accept/send data
        HRESP  = 1'b0;  // OKAY response
        HRDATA = 32'd0;

        // Verify that the PREVIOUS cycle initiated an active transfer
        if (htrans_reg == 2'b10 || htrans_reg == 2'b11) begin // NONSEQ or SEQ
            if (hwrite_reg) begin
                // Write operation: Slave captures HWDATA and loops it back
                HRDATA = HWDATA;
            end else begin
                // Read operation: Send inverted address as dummy data for testing
                HRDATA = ~haddr_reg; 
            end
        end
    end
    
endmodule
How it works: The 1-Cycle Delay

The Register Trap: Notice the always @(posedge HCLK) block. It takes the incoming HADDR and saves it into haddr_reg. It does not act on it immediately. This is the essence of pipelining!

The Execution: In the always @(*) combinational block, the slave checks htrans_reg (the saved state from the previous cycle). If it is a valid write, the slave accepts HWDATA. If it is a valid read, the slave populates HRDATA. For testbench verification purposes, if you ask this slave to read an address, it will reply with the exact opposite (bitwise NOT) of that address.

Zero Wait States: This slave is incredibly fast. It permanently hardcodes HREADY = 1'b1, meaning it never asks the Master to pause.

5. AHB Testbench (`ahb_tb.v`)

The testbench wires everything together. It acts as the "User", filling the Master's FIFO with data, and then commanding the Master to execute three different types of bursts.

`timescale 1ns / 1ps

module ahb_tb();
    
    // TB Signals
    reg clk, rst;
    reg [31:0] data_top;
    reg write_top;
    reg wrap_enb;
    reg enb;
    reg [31:0] addr_top;
    reg [3:0] beat_length;
    
    // AHB Interconnect Wires
    wire hready, hresp;
    wire [31:0] hrdata, HADDR, HWDATA; 
    wire HWRITE;
    wire [2:0] HSIZE, HBURST;
    wire [1:0] HTRANS;
    wire fifo_empty, fifo_full;
    
    master_ahb dut1(
        .CLK_MASTER(clk), .RESET_MASTER(rst), .HREADY(hready), .HRDATA(hrdata),
        .data_top(data_top), .write_top(write_top), .beat_length(beat_length),
        .enb(enb), .addr_top(addr_top), .wrap_enb(wrap_enb),
        .HADDR(HADDR), .HWRITE(HWRITE), .HSIZE(HSIZE), .HBURST(HBURST),
        .HTRANS(HTRANS), .HWDATA(HWDATA), .fifo_empty(fifo_empty), .fifo_full(fifo_full)
    );
    
    slave_ahb dut2(
        .HCLK(clk), .HRESET(rst), .HADDR(HADDR), .HWRITE(HWRITE),
        .HSIZE(HSIZE), .HBURST(HBURST), .HTRANS(HTRANS), .HWDATA(HWDATA),
        .HREADY(hready), .HRESP(hresp), .HRDATA(hrdata)
    );
    
    initial clk = 0;
    always #5 clk = ~clk;
        
    initial begin
        // 1. INITIALIZE SYSTEM
        rst = 1; write_top = 0; data_top = 0; addr_top = 0;
        beat_length = 0; enb = 0; wrap_enb = 0;
        #15 rst = 0; #10;
        
        // 2. FILL THE MASTER'S FIFO (8 Beats of Data)
        @(posedge clk); write_top = 1; data_top = 32'h1111_1111;
        @(posedge clk); write_top = 1; data_top = 32'h2222_2222;
        @(posedge clk); write_top = 1; data_top = 32'h3333_3333;
        @(posedge clk); write_top = 1; data_top = 32'h4444_4444;
        @(posedge clk); write_top = 1; data_top = 32'hAAAA_AAAA;
        @(posedge clk); write_top = 1; data_top = 32'hBBBB_BBBB;
        @(posedge clk); write_top = 1; data_top = 32'hCCCC_CCCC;
        @(posedge clk); write_top = 1; data_top = 32'hDDDD_DDDD;
        @(posedge clk); write_top = 0; 
        #10;
        
        // 3. TRIGGER INCR4 WRITE BURST
        @(posedge clk);
        addr_top = 32'h0000_0040; 
        beat_length = 4;
        wrap_enb = 0; enb = 1; 
        @(posedge clk); enb = 0;
        #80; 

        // 4. TRIGGER WRAP4 WRITE BURST
        @(posedge clk);
        addr_top = 32'h0000_0004; 
        beat_length = 4;
        wrap_enb = 1; enb = 1; 
        @(posedge clk); enb = 0;
        #80; 

        // 5. TRIGGER WRAP4 READ BURST
        @(posedge clk);
        write_top = 0;            
        addr_top = 32'h0000_0018; 
        beat_length = 4;
        wrap_enb = 1; enb = 1; 
        @(posedge clk); enb = 0;
        #100;
        $finish;
    end
endmodule

6. Step-by-Step Waveform Analysis

If you put the testbench above into a simulator, here is exactly what the wave viewer will show. Notice how the Data Phase values always appear one step (one row) below the Address Phase that initiated them.

Scenario 1: INCR4 Write Burst

Conditions: Start Address = 0x0000_0040, HBURST = INCR4, HWRITE = 1.

Beat Address Phase (HADDR) HTRANS Data Phase (HWDATA)
1 0x40 NONSEQ (10) (Data from previous transaction)
2 0x44 SEQ (11) 0x1111_1111 (For address 0x40)
3 0x48 SEQ (11) 0x2222_2222 (For address 0x44)
4 0x4C SEQ (11) 0x3333_3333 (For address 0x48)
- IDLE IDLE (00) 0x4444_4444 (For address 0x4C)

Scenario 2: WRAP4 Write Burst

Conditions: Start Address = 0x0000_0004, HBURST = WRAP4.

Calculation: The base address block starts at 0x00 and ends at 0x10. Because we start at 0x04, we will hit the boundary ceiling at the 4th beat and snap back to 0x00.

Beat Address Phase (HADDR) HTRANS Data Phase (HWDATA)
1 0x04 NONSEQ (10) (Idle Data)
2 0x08 SEQ (11) 0xAAAA_AAAA
3 0x0C SEQ (11) 0xBBBB_BBBB
4 (Wrap) 0x00 SEQ (11) 0xCCCC_CCCC
- IDLE IDLE (00) 0xDDDD_DDDD

Scenario 3: WRAP4 Read Burst

Conditions: Start Address = 0x0000_0018, HBURST = WRAP4, HWRITE = 0.

Calculation: The base block starts at 0x10 and ends at 0x20. The slave generates data by applying a bitwise NOT to the requested address.

Beat Address Phase (HADDR) HTRANS Slave Response (HRDATA)
1 0x18 NONSEQ (10) (Idle Data)
2 0x1C SEQ (11) 0xFFFF_FFE7 (~0x18)
3 (Wrap) 0x10 SEQ (11) 0xFFFF_FFE3 (~0x1C)
4 0x14 SEQ (11) 0xFFFF_FFEF (~0x10)
- IDLE IDLE (00) 0xFFFF_FFEB (~0x14)

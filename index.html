1. Introduction to AMBA AHB

The Advanced Microcontroller Bus Architecture (AMBA) is a set of specifications developed by ARM to standardize communication between different functional blocks of a System-on-Chip (SoC)[cite: 1].

On-Chip Protocols

Act as ultra-fast internal highways connecting the CPU and memory inside a single silicon chip[cite: 1]. They focus on moving massive amounts of data instantly over microscopic distances[cite: 1]. Examples include AHB and AXI[cite: 1].

Peripheral Protocols

Act as external bridges connecting the main chip to outside components like sensors and screens[cite: 1]. They focus on using fewer wires to reliably send data across the physical circuit board[cite: 1]. Examples include I2C, SPI, and USB[cite: 1].

AMBA Family Protocols[cite: 1]

  • AHB (Advanced High-performance Bus): Designed for high-speed, high-bandwidth communication[cite: 1]. It features single clock-edge operation, burst transfers, and a pipelined structure[cite: 1].
  • APB (Advanced Peripheral Bus): Used for connecting low-power, low-bandwidth peripheral devices[cite: 1]. A bridge is required to convert AHB signals to APB signals[cite: 1].
  • AXI (Advanced eXtensible Interface): Used for high-performance, low-latency, and high-bandwidth applications[cite: 1].

2. AHB System Components

The AHB protocol defines the interaction between three main components communicating via a bus (a bundle of signals)[cite: 1]. Role assignment is not permanent; roles can be interchanged as transactions change[cite: 1].

[Insert Diagram: Master, Interconnect/Bus, and Slave connected via bundle of signals]
  1. Managers (Masters): Initiate transactions by issuing read and write commands[cite: 1]. They provide the address, control signals, and data during operations[cite: 1]. To avoid conflicts, only one Manager can control the bus at a time[cite: 1].
  2. Subordinates (Slaves): Respond to the Manager's requests based on the provided address[cite: 1]. Subordinates include memories, peripherals, and bridges (like AHB-to-APB bridges)[cite: 1]. If the manager sends an address that doesn't match any subordinate, the response is invalid[cite: 1].
  3. Interconnects (Decoders/Multiplexors): Handle data routing and arbitration[cite: 1]. A decoder selects the appropriate slave (via HSEL signals) based on the address provided by the Master[cite: 1].

3. Key AHB Signals

AHB operates synchronously on the rising edge of the clock, ensuring predictable timing[cite: 1].

Signal Group Signal Name Description
Global Signals[cite: 1] HCLK System clock. All signal timings are related to the rising edge of HCLK[cite: 1].
HRESETn Active-low reset signal. 0 implies reset, 1 implies no reset[cite: 1].
Address & Control[cite: 1] HADDR[31:0] Address bus carrying the memory or peripheral address to read from or write to[cite: 1].
HWRITE Indicates direction. 1 = Write operation (Master to Slave), 0 = Read operation (Slave to Master)[cite: 1].
HSIZE[2:0] Defines the size of the data item/packet[cite: 1]. Must be less than or equal to the data bus width[cite: 1].
HBURST[2:0] Defines the burst mode (single, incrementing, or wrapping) and the number of data transfers in a single burst[cite: 1].
HTRANS[1:0] Indicates the transfer type (IDLE, BUSY, NONSEQ, SEQ)[cite: 1].
Data Buses[cite: 1] HWDATA[31:0] Write data bus driven by the Master during write operations[cite: 1].
HRDATA[31:0] Read data bus driven by the Slave during read operations[cite: 1].
Transfer Response[cite: 1] HREADY Input for Master, Output from Slave[cite: 1]. 1 = Slave is ready, 0 = Slave not ready (wait state)[cite: 1].
HRESP Status of the transaction. 0 = OKAY, 1 = ERROR[cite: 1].
Note on HSIZE[cite: 1]:
  • 3'b000: 8 bits (Byte)[cite: 1]
  • 3'b001: 16 bits (Half Word)[cite: 1]
  • 3'b010: 32 bits (Word)[cite: 1]

4. Basic Transfers & Phases

The AHB is a pipelined protocol[cite: 1]. A basic transfer consists of two distinct phases[cite: 1]:

  1. Address Phase (1st cycle): Lasts for a single HCLK cycle[cite: 1]. All control signals must be valid, and the address of the first packet is provided[cite: 1].
  2. Data Phase (2nd cycle): The master goes to read or write the data[cite: 1]. This phase might require several HCLK cycles; the HREADY signal is used to control the number of clock cycles needed to complete the transfer[cite: 1].
[Insert Timing Diagram: Pipelined Address and Data Phases]

Wait States (HREADY)

The master checks the readiness of the slave[cite: 1]. If HREADY = 0, the slave is not ready, and the master must insert a wait state[cite: 1]. The data phase is extended[cite: 1].

  • Read Transfer Wait: If HREADY = 0, the slave does not provide valid data on HRDATA until the transfer is about to complete (when HREADY goes to 1)[cite: 1].
  • Write Transfer Wait: If HREADY = 0, the master must hold the data stable on HWDATA throughout the extended cycles[cite: 1].
  • During a waited state, the master cannot change the phase from Data phase to Address phase until HREADY = 1[cite: 1].

5. Transfer Types (HTRANS)

The 2-bit HTRANS[1:0] signal determines the type of transfer occurring on the bus[cite: 1].

HTRANS[1:0] Type Description
00 IDLE No data transfer is required[cite: 1]. Used when a Master does not want to perform a transfer[cite: 1]. Subordinates must provide a zero wait state OKAY response and ignore the transfer[cite: 1]. The master is permitted to change the address during IDLE transfers[cite: 1].
01 BUSY Indicates the master is busy and needs to insert idle cycles in the middle of a burst[cite: 1]. The burst continues, but the next transfer cannot happen immediately[cite: 1]. Subordinates must provide a zero wait state OKAY response[cite: 1]. The master can change from BUSY to any other transfer type when HREADY is LOW[cite: 1].
10 NONSEQ Indicates a single transfer or the first transfer of a burst[cite: 1]. The address and control signals are unrelated to the previous transfer[cite: 1].
11 SEQ Indicates the remaining transfers in a burst[cite: 1]. The address is related to the previous transfer[cite: 1].

6. Burst Operations (HBURST)

Sending multiple data packets is called a burst[cite: 1]. The protocol supports single transfers, bursts of 4, 8, and 16 beats, and undefined length bursts[cite: 1]. It supports both incrementing and wrapping bursts[cite: 1].

HBURST[2:0] Type Description
000SINGLESingle data packet transfer[cite: 1].
001INCRIncrementing burst of undefined length[cite: 1].
010WRAP44-beat wrapping burst[cite: 1].
011INCR44-beat incrementing burst[cite: 1].
100WRAP88-beat wrapping burst[cite: 1].
101INCR88-beat incrementing burst[cite: 1].
110WRAP1616-beat wrapping burst[cite: 1].
111INCR1616-beat incrementing burst[cite: 1].

Incrementing Bursts (INCR)

These bursts access sequential locations[cite: 1]. The address of each transfer in the burst is simply an increment of the previous address (Address of packet = Previous Address + Size)[cite: 1]. We keep increasing until the burst reaches its end[cite: 1].

Wrapping Bursts (WRAP)

These bursts wrap when they cross an address boundary[cite: 1]. After reaching the boundary, the address wraps back to the initial starting address of that boundary block[cite: 1].

Calculating Address Boundaries[cite: 1]:
Address Boundary = Number of Beats (HBURST) × Size of Transfer (HSIZE)[cite: 1].
Example: A 4-beat wrapping burst (WRAP4) of word accesses (4-byte HSIZE) wraps at a 16-byte boundary (4 beats × 4 bytes = 16)[cite: 1]. If the start address is 0x34, the 4 transfers will go to 0x34, 0x38, 0x3C, and then wrap back to 0x30[cite: 1]. In AHB, the starting address cannot be an unaligned address[cite: 1].

7. Subordinate Response (HRESP)

After any transaction, the slave sends its status to the master via the 1-bit HRESP signal[cite: 1].

  • 0 = OKAY: The transfer has either completed successfully, or additional cycles are required for the subordinate to complete the request (pending)[cite: 1].
  • 1 = ERROR: An error has occurred during the transfer, indicating it was unsuccessful[cite: 1]. A two-cycle response is required for an error condition, with HREADY being asserted in the second cycle[cite: 1].
